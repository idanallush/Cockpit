# דשבורד מעקב API וניהול עלויות

## תיאור הפרויקט
דאשבורד ניהול מרכזי שמציג בזמן אמת את כל 5-10 הפרויקטים והכלים שלי, עם מעקב אחר סטטוס חיבורי API (OpenAI, Anthropic, Google APIs), בדיקת תקינות שירותים, וניתוח עלויות מפורט לכל קריאת API. הדאשבורד יכלול טבלאות עם פרטי כל מפתח API, עלות נוכחית ותצרוכת לפי פרויקט, גרפים של שימוש חודשי, והתראות על תקלות או חריגות בעלויות. הנתונים יגיעו משילוב של API calls ישירים לספקים, webhook notifications, ובדיקות health check אוטומטיות.

## סיכום ארכיטקטורה
דשבורד מרכזי שמציג בזמן אמת את סטטוס כל ה-APIs שלך (OpenAI, Anthropic, Google), עלויות נוכחיות, ותצרוכת לפי פרויקט. המערכת בונה על Next.js עם Supabase לאחסון נתונים והיסטוריה, עם Vercel Functions לבדיקות תקינות אוטומטיות ומעקב עלויות.

## סטאק טכנולוגי

- **Frontend**: Next.js + Tailwind + shadcn/ui + Recharts
- **Backend**: Next.js API Routes + Vercel Edge Functions
- **Database**: Supabase PostgreSQL + Realtime
- **Hosting**: Vercel
- **Extras**: Supabase Auth, Upstash Redis (cache), React Query (data fetching)

## שכבות הארכיטקטורה

### ממשק משתמש
דשבורד עם טבלאות ממוינות, גרפים של שימוש וחיווי ויזואלי על סטטוס APIs

- **GitHub** (גיטהאב) [חובה]
  - המקום שבו תשמור את כל הקוד של הדשבורד, כמו תיקייה מאורגנת עם היסטוריה של כל השינויים
  - למה צריך: הפרויקט שלך צריך את זה כי תרצה לעקוב אחר שינויים בקוד ולחזור לגרסה קודמת אם משהו נשבר
- **Deployment** (פריסה) [חובה]
  - תהליך שלוקח את הקוד שלך ועושה ממנו אתר חי שאנשים יכולים לגשת אליו באינטרנט
  - למה צריך: הפרויקט שלך צריך את זה כי אתה רוצה לגשת לדשבורד מכל מכשיר ולא רק מהמחשב שלך

### איסוף נתונים אוטומטי
שליפה תקופתית של נתוני תצרוכת ועלויות מ-APIs השונים

- **Serverless** (ללא שרת) [חובה]
  - קוד שרץ אוטומטית בענן בלי שתצטרך לנהל שרת, כמו שעון מעורר שמפעיל פונקציות
  - למה צריך: הפרויקט שלך צריך את זה כדי לבדוק כל כמה דקות את סטטוס ה-APIs ולעדכן נתוני עלויות אוטומטית
- **API** (API) [חובה]
  - דרך לתוכנות לדבר אחת עם השנייה, כמו ווטסאפ בין מערכות מחשב
  - למה צריך: הפרויקט שלך צריך את זה כדי לשלוף נתונים מ-OpenAI, Anthropic וגוגל ולהציג אותם בדשבורד
- **Webhook** (ווב-הוק) [מומלץ]
  - הודעה אוטומטית שהשירות שולח לך כשקורה משהו חשוב, כמו SMS שמגיע כשיש חיוב חדש
  - למה צריך: הפרויקט שלך צריך את זה כדי לקבל התראות מיידיות כשיש בעיה עם API או עלות חורגת מהתקציב
- **Environment Variables** (משתני סביבה) [חובה]
  - מקום בטוח לשמור סיסמאות ומפתחות סודיים מחוץ לקוד, כמו כספת
  - למה צריך: הפרויקט שלך צריך את זה כדי לשמור בבטחה את מפתחות ה-API שלך ללא חשיפה בקוד

### אחסון והיסטוריה
שמירת נתוני תצרוכת, עלויות והיסטוריה של בדיקות תקינות

- **Database** (מסד נתונים) [חובה]
  - מקום מאורגן לשמור מידע שהאפליקציה צריכה לזכור, כמו ארכיון דיגיטלי
  - למה צריך: הפרויקט שלך צריך את זה כדי לשמור היסטוריה של עלויות ושימוש לבניית גרפים ומעקב טרנדים
- **Cache** (קאש) [מומלץ]
  - זיכרון מהיר שחוסך זמן טעינה על ידי שמירת מידע שכבר נשלף
  - למה צריך: הפרויקט שלך צריך את זה כי בלי זה כל פעם שתרענן את הדשבורד יקח זמן רב לשלוף נתונים מכל ה-APIs

### ניטור ואבטחה
הגנה על מפתחות API והתראות על תקלות או חריגות עלויות

- **OAuth** (OAuth) [חובה]
  - מערכת כניסה בטוחה שמאמתת שאתה באמת זה שטוען להיות, כמו קוד אישי לכספומט
  - למה צריך: הפרויקט שלך צריך את זה כי הדשבורד מכיל מידע רגיש על עלויות ומפתחות API שלא צריך להיות פתוח לכולם
- **API Key** (מפתח API) [חובה]
  - קוד סודי שמזהה אותך מול שירותים חיצוניים, כמו מפתח לדלת
  - למה צריך: הפרויקט שלך צריך את זה כדי לקבל גישה לנתוני תצרוכת ועלויות מ-OpenAI, Anthropic וגוגל
- **SSL/TLS** (SSL/TLS) [חובה]
  - הצפנה שמגינה על המידע בזמן שהוא נוסע ברשת, כמו מעטפה חותמת
  - למה צריך: הפרויקט שלך צריך את זה כדי להגן על מפתחות ה-API ונתוני העלויות הרגישים שלך

## זרימת נתונים

- **Serverless** → **API**: בדיקות סטטוס APIs וקבלת נתוני תצרוכת מהספקים
- **API** → **Database**: שמירת נתוני עלויות והיסטוריית שימוש לפי פרויקט
- **Database** → **Cache**: נתוני דשבורד נוכחיים לתצוגה מהירה
- **Webhook** → **Database**: התראות בזמן אמת על שינויים בסטטוס או עלויות
- **OAuth** → **Deployment**: אימות גישה בטוח לדשבורד

## סדר פיתוח

### שלב 1: תשתית בסיסית (4-6 ימים)
הקמת פרויקט Next.js, חיבור Supabase, מערכת כניסה, והעלאה ראשונית ל-Vercel. בסוף השלב תוכל להתחבר לדשבורד ריק.
- קומפוננטות: GitHub, Database, OAuth, Deployment

### שלב 2: מעקב APIs בסיסי (5-7 ימים)
בניית הלוגיקה לקריאה מ-OpenAI/Anthropic APIs, הצגת נתוני תצרוכת בטבלאות, ושמירת היסטוריה במסד הנתונים.
- קומפוננטות: API, Environment Variables, API Key

### שלב 3: אוטומציה וביצועים (3-5 ימים)
הוספת בדיקות אוטומטיות כל 5 דקות, מערכת קאש לטעינה מהירה, וגרפים בסיסיים של שימוש חודשי.
- קומפוננטות: Serverless, Cache

### שלב 4: התראות והגנות (2-4 ימים)
מערכת התראות על חריגות עלויות, הקשחת אבטחה, ווידוא שכל התקשורת מוצפנת.
- קומפוננטות: Webhook, SSL/TLS

## הערכת עלויות
